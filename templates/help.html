How to Use LogLite

Supported File Formats: .log and .txt

  1. Navigate to the Upload page or click the "Upload Log File" tab on the home page.
  2. Click the "Choose File" button to select a log file from your computer.
  3. Once selected, click the "Analyze" button to begin analysis.
  4. The system will process each log line individually and provide a comprehensive analysis.
  5. Results will appear in the dashboard section showing anomalies, system health score, and detailed breakdowns.
For optimal performance, log files should contain one log entry per line.

  1. Navigate to the home page and click the "Enter Log Text" tab.
  2. Type or paste log entries in the text area (one entry per line for multiple logs).
  3. Click the "Analyze" button to process the logs.
  4. Results will be displayed in the dashboard showing anomalies and system health information.
You can enter multiple log entries, just make sure each log is on a separate line.

Log Analysis Table: Lists each log with its prediction (normal or anomaly type) and confidence level.

Detailed View: Click "View Details" for any log to see:

  • Affected components (memory, storage, network, etc.)
  • Key indicators (important words in the log)
  • Root cause analysis
  • Recommended actions

LogLite provides a RESTful API for integrating with other applications:

Endpoint: /api/logs/analyze

Method: POST

Parameters:

  • log_file: Log file to analyze (multipart/form-data)
  • OR log_text: Log text to analyze (application/x-www-form-urlencoded)

Response: JSON object containing analysis results

Example curl request:

curl -X POST -F "log_text=instruction cache parity error corrected" http://localhost:5000/api/logs/analyze
